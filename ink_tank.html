Project 1: Ink Tank

Ink Tank was created in order to provide people with a mobile platform on which they can easily gain tattoo inspiration, book appointments, and connect with a community of tattoo enthusiasts. It started from the desire to create a solution for people who wanted a tattoo, but was overwhelmed by how to get started.

My Process

IDEATION

The first thing we must do is understand the problem we are trying to find a solution for. Therefore, I started this project by creating a listed of possible problems and solutions that users might face. Next, I created user personas and user stories to better understand the user's perspective. Through our visualized scenarios and user journeys, we begin to see what components we will need to build into our application.

WIREFRAME

In the beginning stages of wireframing, I created rough sketches with paper and pencil– this allows us to quickly create and revise layouts for mobile and desktop versions of the app. Next, I created mid-fidelity wireframes on Balsamiq allowing me to visualize how the app will look on a mobile screen.

PROTOTYPES & ITERATIONS

From mid-fidelity wireframes, I created high-fidelity prototypes for usability testing. Through data collection, we can truly determine whether we have created a useful product. With the insights gained from research, the prototype is continuously improved.

COLLABORATION & REVIEW

Even after iterations based on usability testing, Ink Tank would benefit from other designers’ feedback. I learned a lot by examining others’ projects, listening to feedback, and observing how my peers interacted with the product. This project is still a work in progress as I discover ways in which it could be improved, always open to make changes to ensure we are always designing a solution for users.
